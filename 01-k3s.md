# 1. k3s の設定

## 1.1. Control Plan

```bash
# インストールと構築
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="server --tls-san pc1.local" sh -
```

## 1.2. Worker Node

```bash
# pc2 で実行
# Goではデフォルトでは *.local(mDNS) の名前解決ができないので、アクセスをさせる場合は MulticastDNS を有効にする
sudo nano /etc/systemd/resolved.conf

# MulticastDNS=yes

sudo systemctl restart systemd-resolved

# 127.0.0.53 は Linux に標準搭載されているシステム内部の専属DNS(systemd-resolved)の IP アドレス
nslookup pc1.local 127.0.0.53
# これで名前解決がされていればOK
```

```bash
# pc1 で実行
# Token の取得
sudo cat /var/lib/rancher/k3s/server/node-token
```

```bash
# pc2 で実行
# アクセスできることを確認
nc -zv pc1.local 6443
# pc2 で実行
curl -sfL https://get.k3s.io | K3S_URL=https://pc1.local:6443 K3S_TOKEN=ここにトークンを貼り付け sh -
```

```bash
# pc1 で以下のコマンドを実行してノードが追加されていれば OK
k3s kubectl get nodes
```

Node が問題なく動いているかの確認

```bash
cat <<EOF > test-pod.yaml
apiVersion: v1
kind: Pod
metadata:
  name: test-nginx-on-pc2
spec:
  nodeSelector:
    kubernetes.io/hostname: "pc2"  # ここで PC2 を名指ししています
  containers:
  - name: nginx
    image: nginx:alpine
EOF

k3s kubectl apply -f test-pod.yaml

# これで Pod が立ち上がり Running になっていれば OK
k3s kubectl get pods -o wide

# 後片付け
k3s kubectl delete -f test-pod.yaml
rm test-pod.yaml
```
